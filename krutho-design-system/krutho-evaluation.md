# Krutho Design Evaluation

Last edit 23rd April 2026

---

Every output is built to be inspected. Evaluation is how inspection happens. It is the layer at which finished work is tested against the standard the system holds.

---

## What is evaluated

Foundations and applications are evaluated differently.

Foundations (colour, typography, spacing, grid, layout, and the specifications that govern them) are evaluated by measurable checks. A token is applied or it is not. A derivation holds or it does not. A contrast ratio meets the threshold or it does not. Foundation correctness is inspectable in the formal sense.

Applications (surfaces, compositions, finished work) are evaluated by principled checks. The question is whether the work achieves the purpose it was made for. This is tested, not measured.

---

## The five questions

Finished work is tested against five questions. Each is answerable by inspection.

1. **Does it achieve its goal?** The work was made for a purpose. The purpose was named. The test is whether the work achieves it.

2. **Can it be used without friction?** The audience can engage with the work as intended. Nothing obstructs use.

3. **Does it communicate clearly?** The work's meaning is readable. The audience knows what it is saying without interpretation.

4. **Does it produce the intended register?** The feeling the work is meant to create is the feeling it produces. Register is softer than function but still observable.

5. **Does it hold across contexts?** The work behaves consistently where it appears. Surfaces, devices, audiences, time.

Work that passes all five is good work. Work that fails one is incomplete, and the failure names the correction.

---

## When evaluation runs

Evaluation runs on finished work. The principles orient decisions as they are made. Evaluation tests whether the finished output holds.

Per-decision verification produces friction. Per-output evaluation produces clarity about whether the work meets the standard.
